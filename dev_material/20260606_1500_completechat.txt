❯ I am not sure what R_dot is

● R_dot is just my shorthand for data rate — the bits per second the system has to move off the front end (store or, worse, transmit over BLE). The "dot" is the physics convention for a per-time rate (like ẋ = velocity). So R = data, Ṙ = data per second. I'll rename it to
  data_rate in the actual code — the symbol was needless jargon.

  Where it comes from

   one A-line (one pulse-echo):
     t_acq = 2·D / c          round-trip time to depth D and back
     N     = fs · t_acq       how many samples that window holds
     bits per A-line = N · bits

   now repeat in time and across channels:
     R_dot = N · bits · PRF · nRx
              │     │     │     └─ nRx parallel receive channels
              │     │     └─────── PRF = how many A-lines per second
              │     └───────────── bits per sample (ADC resolution)
              └─────────────────── samples per A-line
          = bits/second the radio (or storage) must carry

  A concrete number (why it matters)

  ┌────────────────────┬─────────────────────────────────┬───────────────────────────────┐
  │      Quantity      │              Value              │             Note              │
  ├────────────────────┼─────────────────────────────────┼───────────────────────────────┤
  │ D (depth)          │ 3 cm                            │ → t_acq = 2·0.03/1540 ≈ 39 µs │
  ├────────────────────┼─────────────────────────────────┼───────────────────────────────┤
  │ fs (RF mode)       │ 40 Msps                         │ full-bandwidth digitization   │
  ├────────────────────┼─────────────────────────────────┼───────────────────────────────┤
  │ N = fs·t_acq       │ ≈ 1560 samples                  │ per A-line                    │
  ├────────────────────┼─────────────────────────────────┼───────────────────────────────┤
  │ bits               │ 12                              │                               │
  ├────────────────────┼─────────────────────────────────┼───────────────────────────────┤
  │ PRF                │ 100 Hz                          │                               │
  ├────────────────────┼─────────────────────────────────┼───────────────────────────────┤

  # ════════════════════════════════════════════════════════════════════════
  #  ModulUS power/battery model — the spine, made executable
  #  f_Tx is the ONE primary knob; fs and resolution are DERIVED (Option B).
  # ════════════════════════════════════════════════════════════════════════

  # ── Constants (provenance flagged) ──────────────────────────────────────
  C_SOUND    = 1540.0    # m/s   soft-tissue speed of sound [used in ModulUS paper; Szabo, Diagnostic US]
  N_CYCLES   = 5         # –     excitation pulse length     [ModulUS exp. setup]
  K_OS       = 4         # –     RF oversampling: fs = K_OS·f_Tx  → 10 MHz→40 Msps
  BWR_FACTOR = 4         # –     analog BW reduction          [ModulUS: 2.9→0.7 MHz mean]

  ADC_FOM       = 50e-15 # J/conv-step  Walden FoM            ⚠ cite TBD (Walden'99 / Murmann survey)
  FS_ONCHIP_MAX = 5e6    # samp/s  STM32L496 on-chip ADC ceiling [ModulUS / ST datasheet]

  E_BIT      = 20e-9     # J/bit  effective BLE energy        ⚠ PLACEHOLDER → measure on NRF53
  R_BLE_MAX  = 1e6       # bit/s  usable BLE throughput        ⚠ cite/measure TBD

  RHO_VOL    = 400.0     # Wh/L   Li-ion volumetric density    ⚠ cite TBD
  RHO_GRAV   = 230.0     # Wh/kg  Li-ion gravimetric density   ⚠ cite TBD
  CR2032_WH  = 0.65      # Wh     reference coin cell          [coin-cell datasheet]
  N_FEATURES = 16        # –      features-mode payload / A-line


  def axial_res(f_Tx, n_cycles=N_CYCLES, c=C_SOUND):
      """Axial resolution = half the spatial pulse length. Depends ONLY on f_Tx
      and pulse length — NOT on mode. (BWR keeps resolution; it only drops fs.)"""
      lam = c / f_Tx                       # wavelength [m]
      return n_cycles * lam / 2.0          # [m]


  def sample_rate(f_Tx, mode):
      """Derived, not chosen. RF digitizes the full echo; BWR/features digitize
      the analog envelope at 1/BWR_FACTOR the rate (the ModulUS trick)."""
      fs_rf = K_OS * f_Tx                  # full-bandwidth Nyquist-with-margin
      return fs_rf if mode == "RF" else fs_rf / BWR_FACTOR


  def system_power(f_Tx, bits, nRx, PRF, D, mode,
                   c=C_SOUND, FOM=ADC_FOM, E_bit=E_BIT,
                   P_afe_on=3e-3, P_afe_leak=50e-6,
                   P_pulser=1e-3, P_idle=1e-3, P_feat=2e-3,
                   n_features=N_FEATURES):
      fs     = sample_rate(f_Tx, mode)
      t_acq  = 2 * D / c                    # round-trip window [s]
      N      = fs * t_acq                   # samples per A-line
      duty   = t_acq * PRF                  # active fraction

      if mode == "features":
          data_rate = n_features * bits * PRF * nRx     # decoupled from N
      else:                                              # RF or BWR (differ via fs→N)
          data_rate = N * bits * PRF * nRx

      P_adc   = FOM * (2**bits) * fs * duty * nRx        # Walden, ENOB≈bits (1st order)
      P_afe   = P_afe_on * duty * nRx + P_afe_leak
      P_radio = data_rate * E_bit                        # usually the winner when streaming
      P_mcu   = P_idle + (P_feat if mode == "features" else 0.0)
      P_avg   = P_pulser + P_adc + P_afe + P_radio + P_mcu

      return dict(
          fs=fs, N=N, duty=duty, data_rate=data_rate,
          axial_res_mm=axial_res(f_Tx, c=c) * 1e3,
          P_adc=P_adc, P_afe=P_afe, P_radio=P_radio,
          P_mcu=P_mcu, P_pulser=P_pulser, P_avg=P_avg,
          fits_ble    = data_rate <= R_BLE_MAX,          # wall 1: the radio
          fits_onchip = fs <= FS_ONCHIP_MAX,             # wall 2: the ADC → MCU-vs-FPGA
      )


  def battery(P_avg, days=1, rho_vol=RHO_VOL, rho_grav=RHO_GRAV):
      Wh = P_avg * 86400 * days / 3600.0
      return dict(Wh=Wh, vol_cm3=Wh / rho_vol * 1000.0,
                  mass_g=Wh / rho_grav * 1000.0,
                  n_cr2032=Wh / CR2032_WH)               # battery in "coin-cell units"


  def fom_mw_per_mhz(P_avg_mW, nRx, f_Tx_Hz):
      """Paper's figure of merit: avg power per Rx channel, normalized by f_Tx.
      Lets the live ModulUS number land near 0.6 mW/MHz (Table I)."""
      return P_avg_mW / nRx / (f_Tx_Hz / 1e6)

  The convention (lean, uniform — no ABC ceremony)

  A small shared acquisition context carries the derived chain so blocks don't recompute or pass 8 args around (DRY):

  @dataclass
  class Acq:                       # the running state as we walk the stack
      f_Tx: float; bits: int; nRx: int; PRF: float; D: float; mode: str
      fs: float = 0.0              # filled by ADC
      N: float = 0.0              # filled by ADC
      duty: float = 0.0           # filled by Pulser
      data_rate: float = 0.0      # filled by ADC/MCU

  Each block is a dataclass of its specs + a power(acq) method; signal-touching blocks also expose a process(trace) for the real-data path. Example — the AFE, the block that earns the paper:

  @dataclass
  class AFE:
      """ModulUS analog front-end. RF mode = passthrough; BWR mode = envelope
      detect, which collapses the bandwidth (paper: 2.9 -> 0.7 MHz)."""
      bwr_factor: int = 4
      p_on: float = 3e-3          # W, active   [in-sandbox: datasheet/measure]
      p_leak: float = 50e-6       # W, leakage

      def bandwidth_factor(self, mode):        # how much it eases the ADC
          return 1 if mode == "RF" else self.bwr_factor

      def process(self, rf, fs):               # real-signal twin (Act 1)
          return envelope(rf, fs)              # |Hilbert| or analog model

      def power(self, acq):
          return self.p_on * acq.duty * acq.nRx + self.p_leak

  @dataclass
  class ADC:
      """STM32L496 on-chip ADC: dual 5 Msps, 12-bit. The 5 Msps cap is the wall."""
      fs_max: float = 5e6
      fom: float = 50e-15                       # Walden  [cite TBD]

      def configure(self, acq, afe):            # derive fs from transducer + AFE
          nyq = 2 * acq.f_Tx
          acq.fs = nyq / afe.bandwidth_factor(acq.mode)
          acq.N  = acq.fs * (2 * acq.D / 1540.0)
          return acq.fs <= self.fs_max          # fits_onchip?

      def power(self, acq):
          return self.fom * (2**acq.bits) * acq.fs * acq.duty * acq.nRx

  How System composes them = the spine, walked in physical order

  class System:
      def __init__(self):
          self.tx, self.pulser, self.afe = Transducer(), Pulser(), AFE()
          self.adc, self.mcu, self.radio = ADC(), MCU(), Radio()

      def walk(self, acq):                       # signal flows down the stack
          self.pulser.configure(acq)             # duty
          fits_adc = self.adc.configure(acq, self.afe)   # fs, N
          self.mcu.configure(acq)                # data_rate (features vs full)
          fits_ble = self.radio.fits(acq)
          P = {b.__class__.__name__: b.power(acq)
               for b in (self.pulser, self.afe, self.adc, self.mcu, self.radio)}
          return dict(acq=acq, P=P, P_avg=sum(P.values()),
                      fits_onchip=fits_adc, fits_ble=fits_ble)

   27 N_FEATURES = 16        # -      features-mode payload per A-line   #

   30 FS_ONCHIP  = 5e6       # samp/s  STM32L496 on-chip ADC ceiling    # [ST datasheet]

   40 # ── Acquisition context: state filled as the signal walks the stack ───────
   41 @dataclass
   42 class Acq:
   43     f_Tx: float; bits: int; nRx: int; PRF: float; D: float; mode: str
   44     fs: float = 0.0          # filled by Core.acquire
   45     N: float = 0.0           # filled by Core.acquire
   46     duty: float = 0.0        # filled by Ping.configure

   50     def t_acq(self):

   52
   53

   55 @dataclass
   56 class Transducer:
   57     n_cycles: int = N_CYCLES
   58
   59     def axial_res(self, f_Tx):

   61         return self.n_cycles * lam / 2.0 # half the spatial pulse length [m]
   62

   64 # ── Ping = pulser board (STHVUP32 + T/R switch): send the pulse ───────────

   72
   73     def power(self, acq):

   75
   76
   77 # ── Echo = AFE board (envelope detector): catch & condition the echo ──────
   78 @dataclass
   79 class Echo:
   80     bwr_factor: int = BWR_FACTOR

   82     p_leak: float = 50e-6               # W leakage  #

   84     def bandwidth_factor(self, mode):

   86
   87     def process(self, rf):              # real-signal twin: |Hilbert| envelope
   88         from scipy.signal import hilbert
   89         return np.abs(hilbert(rf))
   90
   91     def power(self, acq):
   92         return self.p_on * acq.duty * acq.nRx + self.p_leak
   93
   94
   95 # ── Core = control board (STM32 + dual ADC): digitize, sequence, compute ──
   96 @dataclass
   97 class Core:

  102     n_features: int = N_FEATURES

  104     def acquire(self, acq, echo):       # derive fs, N from transducer + Echo
  105         nyq = K_NYQUIST * acq.f_Tx
  106         acq.fs = nyq / echo.bandwidth_factor(acq.mode)
  107         acq.N  = acq.fs * acq.t_acq
  108         return acq.fs <= self.fs_max    # fits on-chip ADC? (else needs external/FPGA)
  109
  110     def compute(self, acq):             # MCU decides the payload to ship
  111         if acq.mode == "features":
  112             acq.data_rate = self.n_features * acq.bits * acq.PRF * acq.nRx
  113         else:                           # RF / BWR differ only via fs -> N
  114             acq.data_rate = acq.N * acq.bits * acq.PRF * acq.nRx
  115
  116     def power(self, acq):
  117         p_adc = self.fom * (2**acq.bits) * acq.fs * acq.duty * acq.nRx
  118         p_mcu = self.p_idle + (self.p_feat if acq.mode == "features" else 0.0)
  119         return p_adc + p_mcu
  120
  121
  122 # ── External downstream: BLE radio (NOT a ModulUS board) ──────────────────
  123 @dataclass
  124 class Radio:
  125     e_bit: float = E_BIT
  126     r_max: float = R_BLE_MAX
  127
  128     def fits(self, acq):
  129         return acq.data_rate <= self.r_max
  130
  131     def power(self, acq):
  132         return acq.data_rate * self.e_bit

  135 # ── External power source: battery sizing ─────────────────────────────────
  136 @dataclass

  139     rho_grav: float = RHO_GRAV

  143         return dict(Wh=Wh, vol_cm3=Wh / self.rho_vol * 1000.0,
  144                     mass_g=Wh / self.rho_grav * 1000.0, n_cr2032=Wh / CR2032_WH)

  148     return P_mW / nRx / (f_Tx / 1e6)

  Why Scheme A (Ping / Echo / Core):
  - Pedagogical: the board names are the A-mode lesson — ping out, echo back, core processes. Students can't forget the signal chain because the boards spell it.
  - Memorable + brandable: one syllable each, ecosystem-style (like HAT/Cape), reads as a product line.
  - Accurate: "ping" is the genuine sonar/pulse-echo term; "echo" is exactly what the AFE conditions.
  - Clean software twins: Ping, Echo, Core, Motherboard/System.

  Scheme B (Drive/Sense/Core) is the safe self-documenting fallback if you find Ping/Echo too playful for an IEEE context — though I'd argue Ping/Echo is technical, not cute, since it names the physics.
